# 2D 矩形寻路算法演示项目

高性能 2D 矩形障碍物动态寻路系统，支持大地图（50000×50000）和大量障碍物（150静态 + 20动态）的实时路径规划。

## 项目特点

### 核心算法
- **Minkowski 和扩展** - 将矩形移动者简化为点，障碍物扩展等效尺寸
- **可视图 (Visibility Graph)** - 基于障碍物角点构建可见性网络
- **A* 搜索** - 启发式最短路径算法
- **JPS (Jump Point Search)** - 正方形移动者的跳点搜索优化
- **路径平滑** - 贪心消除冗余拐点

### 性能优化
- ✅ **静态可见性图预计算** - 静态障碍物角点可见性一次性预计算，寻路时 O(1) 查表
- ✅ **空间哈希加速** - 碰撞检测和邻居查询从 O(n) 优化到 O(1)~O(k)
- ✅ **可见性缓存** - 单次寻路内复用可见性检测结果
- ✅ **内存池** - 高频对象复用，降低 GC 压力
- ✅ **Lazy A*** - 按需计算可见边，避免 O(n²) 预计算

### 规模支持
- 地图大小：**50000 × 50000**
- 静态障碍物：**150 个**
- 移动障碍物：**20 个**（实时动态避障）
- 寻路耗时：平均 **0.5-3ms**，缓存命中 **< 0.01ms**

- 150 个随机静态障碍物（250-2050 尺寸）
- 20 个正方形移动障碍物（32/64/128/256 尺寸）

## 算法详解

### 1. Minkowski 扩展

将矩形移动者简化为点，障碍物在各方向扩展移动者的半尺寸：

```
移动者: 64×64 (半径 32)
障碍物: 800×600

扩展后: (800+64)×(600+64) = 864×664
```

### 2. 可视图构建

提取扩展后障碍物的角点（向外偏移 0.5 避免擦边）：

```
每个障碍物 → 4个角点
150个静态障碍物 → ~600个静态角点
20个动态障碍物 → ~80个动态角点

总节点数 = 起点 + 终点 + 静态角点 + 动态角点
```

### 3. 静态图预计算

**初始化时：**
1. 计算所有静态角点间的可见性（~600×600 = 360,000 次检测）
2. 构建邻接表存储可见边（~8,000-12,000 条边）
3. 按移动者尺寸缓存（32/64/128/256）

**寻路时：**
1. 静态-静态：直接查邻接表 O(1)
2. 起点/终点/动态 ↔ 静态：实时计算（~80×600 = 48,000 次）
3. 动态-动态：实时计算（~80×80 = 6,400 次）

**优化效果：**
- 预计算前：360,000 + 48,000 + 6,400 = **414,400 次检测**
- 预计算后：48,000 + 6,400 = **54,400 次检测**
- **减少 87%** 的可见性检测

### 4. A* 搜索

启发式函数：`f(n) = g(n) + h(n)`
- `g(n)` = 起点到当前节点的实际距离
- `h(n)` = 当前节点到终点的欧几里得距离（保证可采纳）

### 5. 路径平滑

贪心跳过冗余节点：

```
原始路径: A → B → C → D → E
检测: A 能直达 E？是 → 简化为 A → E
检测: A 能直达 D？否 → 保留 A → B
继续从 B 开始...
```

## 常见问题

### Q1: 移动障碍物碰撞后停止不动？
**A:** 已修复。问题原因是等待中的移动障碍物被错误地完全忽略，导致寻路时路径穿过它们。现已正确处理等待状态的障碍物。

### Q2: 寻路耗时突然飙升？
**A:** 可能原因：
- 超长距离寻路（>30000）
- 障碍物极度密集区域
- 可视图节点数过多（>2000）

查看 `/api/slowpaths` 获取详细分析。

### Q3: 如何调整障碍物数量？
**A:** 修改 `mapconfig.go` 中 `generateDefaultConfig()` 函数：

```go
// 150 个静止障碍物
for i := 0; i < 150; i++ {  // ← 修改这里
    // ...
}

// 20 个移动障碍物
for i := 0; i < 20; i++ {  // ← 修改这里
    // ...
}
```

### Q4: 如何修改地图大小？
**A:** 修改配置文件或 `generateDefaultConfig()`：

```go
const mapW, mapH = 50000.0, 50000.0  // ← 修改这里
```

### Q5: 为什么初始化需要几百毫秒？
**A:** 静态可见性图预计算需要检测约 360,000 对节点的可见性。这是一次性开销，后续所有寻路都会受益。


```go
const (
    epsilon      = 1e-9   // 浮点精度

)
```


```go
if logEntry.TimeMs > 0.1 {  // 原 0.3ms，改为 0.1ms
```
